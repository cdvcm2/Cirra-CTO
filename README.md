# cirra-cto
**Cirra Edge — Group CTO Repository**
Version 1.0 | March 2026

This is the single source of truth for all engineering standards across Cirra Edge.

## Who Owns This Repo

**Group CTO only.**

Sub-CTOs (never-2-far, liftcore, lift-staff) are consumers of this repo — never contributors.
No sub-CTO has write access. No exceptions. Ever.

If a sub-CTO identifies a pattern or learning worth promoting to group level, they surface it in their 3-5 day report to the Group CTO. The Group CTO decides, drafts, and commits. The sub-CTO does not touch this repo.

## What Lives Here

- THE_CIRRA_WAY.md — Universal engineering doctrine
- PATTERNS.md — Proven reusable solutions, tagged by stack
- LEARNINGS.md — Cross-project failure log, append-only
- STACK_DECISIONS.md — Why each tool was chosen
- PROMPTS.md — Curated prompt library
- SUB_CTO_REPORT_TEMPLATE.md — What each sub-CTO sends every 3-5 days
- GROUP_CTO_INTAKE_SOP.md — How Group CTO processes reports
- sub-cto-briefs/ — Per-project Claude Project setup briefs

## How Sub-CTOs Use This Repo

1. Upload THE_CIRRA_WAY.md, PATTERNS.md, PROMPTS.md into their Claude Project at setup
2. Pull updates whenever Group CTO pushes a new version
3. Apply patterns tagged for their stack automatically
4. Never modify. Never push. Read and apply.
